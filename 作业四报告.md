## 任务简介

图文检索是典型的跨模态检索任务，目标是在图像和自然语言文本之间建立统一语义空间。给定一张图像，模型需要从候选文本中找出与该图像匹配的描述；给定一段文本，模型需要从候选图像中找出对应图像。

本实验包含两个方向：

- 图到文检索（Image-to-Text, I2T）：给定图像，检索匹配文本。
- 文到图检索（Text-to-Image, T2I）：给定文本，检索匹配图像。

形式化地，设图像集合为 $I=\{i_1,\dots,i_N\}$，文本集合为 $T=\{t_1,\dots,t_M\}$。模型分别通过图像编码器 $f_I(\cdot)$ 和文本编码器 $f_T(\cdot)$ 得到归一化特征，并用相似度

$$
s(i,t)=f_I(i)^\top f_T(t)
$$

对候选样本排序。I2T 任务是在固定图像 $i$ 的条件下对全部文本 $t$ 排序，T2I 任务是在固定文本 $t$ 的条件下对全部图像 $i$ 排序。

评价指标采用 Recall@K，统计正确匹配结果是否出现在排序前 K 个候选中。本文报告 `R@1`、`R@5`、`R@10`，并用 `RSum` 表示六个 Recall 指标之和。

## 方法简介

CLIP 通过大规模图文对比学习训练图像编码器和文本编码器，使匹配图文在同一特征空间中具有更高相似度。推理时，分别编码图像和文本，再通过归一化特征的内积计算相似度。

Long-CLIP 针对 CLIP 文本长度受限的问题进行改进。原始 CLIP 的文本输入长度上限为 77 个 token，难以处理更长、更细粒度的文本描述。Long-CLIP 在保留 CLIP 原有语义能力的基础上，将最大文本长度扩展到 248，并通过知识保持的位置编码扩展和 CLIP 特征主成分匹配，使模型在长文本输入场景下保持更好的图文对齐能力。

复现过程中遇到的问题及解决方式：

1. `hf` 命令不在当前环境 PATH 中，改用 `aria2c` 从 HuggingFace 镜像直链断点续传下载 LongCLIP-B 权重。

3. 官方代码依赖 `pkg_resources`，而当前 `setuptools 82` 不再提供该模块，因此将 `setuptools` 调整到 `<81` 的兼容版本。
4. 官方 retrieval 脚本直接打印结果，不便于保存和对比。复现时实现了统一 evaluator，使 CLIP 与 LongCLIP-B 共用同一份数据读取和 Recall@K 计算逻辑，并输出 JSON 结果。

## 分析与讨论

从复现结果看，LongCLIP-B 在 Flickr30k test split 上明显优于 CLIP ViT-B/32。图到文方向的 R@1 从 78.60 提升到 85.20，文到图方向的 R@1 从 58.12 提升到 70.68，RSum 从 503.16 提升到 538.54。这说明 Long-CLIP 在保持 CLIP 图文对齐能力的基础上，进一步增强了图文检索性能。

Flickr30k 的 caption 通常较短，不能完全体现 Long-CLIP 对长文本输入的优势。即便如此，LongCLIP-B 仍然取得更高召回率，说明其改进不仅对长文本有效，也能迁移到传统短文本图文检索任务。文到图方向提升更明显，可能是因为文本查询需要在 1000 张候选图像中定位唯一图像，对文本编码质量和细粒度语义区分能力更敏感。

本次复现只覆盖论文主实验中的 Flickr30k 图文检索对比，没有进行额外扩展实验。后续如果继续扩展，可以进一步加入长 caption 数据集或按文本长度分组，分析 Long-CLIP 在长文本条件下的收益。
