## 任务简介

图文检索是典型的跨模态检索任务，目标是在图像和自然语言文本之间建立统一语义空间。给定一张图像，模型需要从候选文本中找出与该图像匹配的描述；给定一段文本，模型需要从候选图像中找出对应图像。因此，任务可以分为图到文检索（Image-to-Text, I2T）和文到图检索（Text-to-Image, T2I）两个方向。

形式化地，设图像集合为 $I=\{i_1,\dots,i_N\}$，文本集合为 $T=\{t_1,\dots,t_M\}$。模型分别通过图像编码器 $f_I(\cdot)$ 和文本编码器 $f_T(\cdot)$ 得到归一化特征，并用相似度

$$
s(i,t)=f_I(i)^\top f_T(t)
$$

对候选样本排序。I2T 任务是在固定图像 $i$ 的条件下对全部文本 $t$ 排序，T2I 任务是在固定文本 $t$ 的条件下对全部图像 $i$ 排序。评价指标采用 Recall@K，统计正确匹配结果是否出现在排序前 K 个候选中。本文报告 `R@1`、`R@5`、`R@10`，并用 `RSum` 表示六个 Recall 指标之和。

## 方法简介

CLIP 通过大规模图文对比学习训练图像编码器和文本编码器，使匹配图文在同一特征空间中具有更高相似度。推理时，分别编码图像和文本，再通过归一化特征的内积计算相似度。传统 CLIP 类模型虽然在零样本分类和图文检索中表现较强，但文本编码器存在输入长度限制，且训练语料中短文本占比较高，因此对长描述中的细粒度属性、多个目标关系和空间关系建模不足。

Long-CLIP 的创新点在于不重新训练一个大型语言编码器，而是在 CLIP 框架上扩展其长文本理解能力。首先，它通过知识保持的位置编码扩展，将文本输入长度从原始 CLIP 的短上下文扩展到更长上下文，同时尽量保留低位置 token 已学到的表示能力。其次，它引入 CLIP 特征主成分匹配，在对齐长文本细节信息的同时保留图像中的主要语义成分，避免模型只追求细节而损害原有短文本检索能力。相比直接微调或简单拉长位置编码，Long-CLIP 的优点是改动轻量、可直接替换 CLIP、同时兼顾长文本理解和传统短文本检索效果。

数据集使用 Flickr30k test split，共 1000 张图像和 5000 条 caption。标注文件复用本机已有的 `/mnt/kxh/smx/homework/h3/datasets/flickr30k/annotations.jsonl`，每张图像对应 5 条英文描述。

运行命令如下：

```bash
cd /mnt/kxh/smx/homework/h4
CUDA_VISIBLE_DEVICES=6 bash scripts/run_full.sh
```

复现过程中遇到的问题及解决方式：

1. 模型权重下载与环境依赖存在兼容问题。当前环境中 `hf` 命令不可用，且官方 Long-CLIP 代码依赖 `ftfy` 和旧版 `pkg_resources` 接口。解决方式是使用 HuggingFace 镜像直链和 `aria2c` 断点续传下载 LongCLIP-B 权重，安装缺失的 `ftfy`，并将 `setuptools` 调整到兼容版本。
2. 官方 retrieval 脚本主要面向直接运行，结果以打印为主，不方便与 CLIP baseline 做统一对比。解决方式是实现统一 evaluator，使 CLIP ViT-B/32 与 LongCLIP-B 共用同一份 Flickr30k 标注、相同的特征归一化方式和 Recall@K 计算逻辑，并将结果保存为 JSON，便于复查。

实验结果如下：

| 方向 | 指标 | CLIP ViT-B/32 | LongCLIP-B |
| --- | --- | ---: | ---: |
| 图到文 | R@1 | 78.60 | 85.20 |
| 图到文 | R@5 | 95.40 | 98.50 |
| 图到文 | R@10 | 98.40 | 99.20 |
| 文到图 | R@1 | 58.12 | 70.68 |
| 文到图 | R@5 | 83.18 | 90.36 |
| 文到图 | R@10 | 89.46 | 94.60 |
| 汇总 | RSum | 503.16 | 538.54 |

## 分析与讨论

从复现结果看，LongCLIP-B 在 Flickr30k test split 上明显优于 CLIP ViT-B/32。图到文方向的 R@1 从 78.60 提升到 85.20，文到图方向的 R@1 从 58.12 提升到 70.68，RSum 从 503.16 提升到 538.54。这说明 Long-CLIP 在保持 CLIP 图文对齐能力的基础上，进一步增强了图文检索性能。

Flickr30k 的 caption 通常较短，不能完全体现 Long-CLIP 对长文本输入的优势。即便如此，LongCLIP-B 仍然取得更高召回率，说明其改进不仅对长文本有效，也能迁移到传统短文本图文检索任务。文到图方向提升更明显，可能是因为文本查询需要在 1000 张候选图像中定位唯一图像，对文本编码质量和细粒度语义区分能力更敏感。

本次复现只覆盖论文主实验中的 Flickr30k 图文检索对比，没有进行额外扩展实验。后续如果继续扩展，可以进一步加入长 caption 数据集或按文本长度分组，分析 Long-CLIP 在长文本条件下的收益。
